public interface ISceneManager
{

## 五、模块开发顺序建议

```
Phase 0: Core (02_核心数据结构)
         ├─ Agent-A: GameConfig, PlayerState, CurveData 等数据结构
         └─ 输出: Core.Data 和 Core.Interfaces 程序集

Phase 1: 独立模块（可完全并行）
         ├─ Agent-1: SceneManager (依赖: Core)
         ├─ Agent-2: PlayerManager (依赖: Core)
         ├─ Agent-3: MathParser (依赖: Core)
         └─ Agent-6: UIManager (依赖: Core)

Phase 2: 依赖模块
         ├─ Agent-4: ShootingSystem (依赖: Core, SceneManager, PlayerManager, MathParser)
         └─ Agent-5: TurnManager (依赖: Core, SceneManager, PlayerManager, ShootingSystem)

Phase 3: 集成测试
         └─ 所有模块组合到 GameScene.unity

Phase 4: V2 (独立模块)
         └─ Agent-7: AI (依赖: MathParser, PlayerManager)

Phase 5: V3 (独立模块)
         └─ Agent-8: LocalMultiplayer (依赖: TurnManager, UIManager)

Phase 6: V4 (独立模块)
         └─ Agent-9: Network (依赖: 所有模块, 新增Netcode)
```

## 六、并行开发的Git分支策略

```
main ──────────────────────────────────────────────►
  │
  ├── feature/core-data        (Agent-A: 数据结构)
  ├── feature/scene-manager    (Agent-1: 场景)
  ├── feature/player-manager   (Agent-2: 玩家)
  ├── feature/math-parser      (Agent-3: 数学)
  ├── feature/shooting         (Agent-4: 射击)
  ├── feature/turn-manager     (Agent-5: 回合)
  ├── feature/ui               (Agent-6: UI)
  ├── feature/ai-v2            (Agent-7: V2)
  ├── feature/local-multi-v3   (Agent-8: V3)
  └── feature/network-v4       (Agent-9: V4)
```

每个Agent在自己的分支上开发，通过PR合并到main。
共享代码(Core)由Agent-A维护，其他Agent通过接口依赖。
